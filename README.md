# СОГЛАШЕНИЕ ОБ УРОВНЕ ОБСЛУЖИВАНИЯ ДЛЯ УСЛУГ

## Настоящее Соглашение об уровне обслуживания для Услуг SAP HANA Enterprise Cloud («Услуги HEC») и SAP S/4HANA Cloud, single tenant edition («Услуги STE») (по отдельности именуемые «Облачная услуга»), устанавливает применимые Уровни обслуживания для Услуг HEC, Услуг STE и Предоставления серверов, на которые подписан Заказчик по условиям соответствующего Договора с SAP.

### 1. Определения

Термины, употребленные в этом документе с заглавной буквы, но не определенные в нем, имеют значение, указанное в Соглашении.

- «Согласованный простой» — это любой Простой, запрошенный SAP или Заказчиком и взаимно согласованный сторонами.
- «Рабочий день» — это любой день с понедельника по пятницу, кроме официальных праздников.
- «Вычислительная среда» — устройства ЦОД, серверы, сетевое оборудование, операционные системы и механизмы хранения информации, предоставляемые и используемые SAP для оказания Облачных услуг Заказчику; включает Продуктивную вычислительную среду и любые прочие Вычислительные среды, используемые в непродуктивных целях, которые могут включать среды разработки, контроля качества и тестовую среду.
- «Простой» — это Итоговое количество минут в месяц, которое Услуга HEC или Услуга STE (или Серверы для Предоставления серверов) не отвечают на запросы из Точки демаркации SAP в центре обработки данных, предоставляющем Услугу HEC или Услугу STE (или Серверы для Предоставления серверов), исключая Согласованные простои.
- «Аварийный простой» — это простой во время развертывания критически важных исправлений и обновлений операционной системы.
- «Исключенное время простоя» имеет значение, установленное в пункте 2 ниже.
- «Инцидент» — это незапланированное прерывание услуг или существенное снижение их качества, о котором сообщили Авторизованные пользователи.
- «Период реагирования на инцидент» — это период (в часах или минутах) с момента получения сообщения Заказчика об инциденте службой услуг SAP по сопровождению первого уровня до первого действия, которое сотрудник службы услуг SAP по сопровождению, знакомый со средой Заказчика, предпринял для устранения Инцидента.
- «Лицензионное программное обеспечение» — это приложения, базы данных, программное обеспечение, инструменты и компоненты, принадлежащие Заказчику или приобретенные им по лицензии (за исключением Программного обеспечения по подписке), которые Заказчик предоставляет SAP как часть Вычислительной среды.
- «Местное время» — это часовой пояс в основном месте доступа Заказчика.
- «Месяц» — это календарный месяц.
- «Ежемесячная вознаграждение за услуги» — это вознаграждение (или 1/12 ежегодного вознаграждения ), выплачиваемая за Услуги HEC или Услуги STE (в зависимости от обстоятельств), доступность которых не соответствует Соглашению об уровне обслуживания по Доступности системы.
- «Плановый простой» имеет значение, установленное в пункте 2 ниже.
- «Зачетная неустойка по уровню обслуживания» — это неустойка, рассчитываемая согласно процедуре, описанной в пунктах 2 и 4(а) настоящего Соглашения об уровне обслуживания.
- «Программное обеспечение по подписке» для Услуг HEC имеет значение, установленное в Дополнении, а для Услуг STE означает программное обеспечение SAP, предоставляемое SAP совместно с Услугами STE.
- «Система» — это один или несколько взаимосвязанных и взаимозависимых компонентов, таких как базы данных, серверы, сети, средства балансирования нагрузки, веб- диспетчеры, тестовые среды и т. д., которые в совокупности используются для управления уровнем. Каждое сочетание компонентов, используемое на каждом уровне, эквивалентно одной Системе. 
- «Итоговое количество минут в месяц» измеряется 24 часа 7 дней в неделю в течение Месяца.
- «UTC» — это всемирное координированное время.

### 2. Доступность системы

Уровень обслуживания по Доступности системы для Услуг HEC и Услуг STE («Соглашение об уровне обслуживания по Доступности системы») определяет Доступность системы применимо к Вычислительной среде (и Серверу для Предоставления серверов). Соглашение об уровне обслуживания по Доступности системы вступает в силу после передачи Системы Заказчику.

Соглашение об уровне обслуживания по Доступности системы неприменимо к Лицензионному программному обеспечению, лицензированному Заказчиком у третьих лиц.

«Доступность системы» для каждой Системы рассчитывается следующим образом:

Процент доступности системы = [Итоговое количество минут в месяц — Простой / Итоговое количество минут в месяц] ∗ 100

| Уровень обслуживания | Зачетная неустойка по уровню обслуживания |
|----------------------|---------------------------------------------|
| Продуктивные: 99,5% Доступности системы | Подписка на HEC и Услуги STE: 2% от Ежемесячного вознаграждения за услуги за каждый 1% меньше Соглашения об уровне обслуживания по Доступности системы |
| Непродуктивные: 95,0% Доступности системы | HEC Cloud Start и HEC BYOL: 2% от Ежемесячного вознаграждения за услуги за каждый 0,1% меньше Соглашения об уровне обслуживания по Доступности системы |
| Предоставление серверов: 99,5% Доступности системы | Предоставление серверов (IaaS Basic): 1500 евро в Месяц в совокупности за все случаи меньше Соглашения об уровне обслуживания по Доступности системы |

1. 99,7% Доступности системы или 99,9% Доступности системы для Продуктивной вычислительной среды применяется в том случае, если приобретено Заказчиком.

2. С учетом максимальных ежемесячных сумм Зачетных неустоек по уровню обслуживания, указанных в пункте 4 ниже.

3. Также применяется к S/4HANA CPO и S/4HANA CPE

## Примеры факторов, не зависящих от воли SAP

- (a) Невыполнение Заказчиком своих обязательств (включая заказ обслуживания Лицензионного программного обеспечения, использование версии или выпуска Лицензионного программного обеспечения и/или Программного обеспечения по подписке, находящегося на обслуживании).
- (b) Простой, вызванный причинами, зависящим от Заказчика.
- (c) Прерывание операций в результате требований, выдвигаемых сторонними производителями Лицензионного программного обеспечения.
- (d) Прерывание или остановка Вычислительной среды или ее частей (или Серверов для Предоставления серверов), вызванные качеством Лицензионного программного обеспечения, предоставленного Заказчиком, и/или обусловленные изменениями, которые Заказчик внес в Лицензионное программное обеспечение или Вычислительную среду (или Серверы для Предоставления серверов), кроме случаев, когда это является сферой ответственности SAP по настоящему Соглашению.
- (e) Периоды восстановления данных пользователей (восстановление базы данных с резервного носителя), вызванные по причинам, не зависящим от SAP.

## Приоритеты Инцидентов

- (a) Очень высокий: инциденту присваивается очень высокий приоритет, если инцидент имеет очень серьезные последствия для нормального проведения бизнес-процессов и ИТ-процессов, связанных с основными бизнес-процессами, и требуется выполнение срочных работ.
- (b) Высокий: инциденту присваивается высокий приоритет, если инцидент серьезно затрагивает нормальное выполнение бизнес-процессов.
- (c) Средний: инциденту присваивается средний приоритет, если инцидент затрагивает нормальное выполнение бизнес-процессов.
- (d) Низкий: инциденту присваивается низкий приоритет, если проблема оказывает незначительное влияние или вообще не влияет на нормальное выполнение бизнес-процессов.

## 3. Отчеты об Уровнях обслуживания

SAP отслеживает Уровни обслуживания, обозначенные в настоящем договоре, и сообщает о них Заказчику в ежемесячном сводном отчете.

- (a) Заказчик должен уведомить SAP о любых заявках на Зачетных неустойках по уровню обслуживания в течение 1 (одного) месяца после получения ежемесячного отчета о Доступности системы.
- (b) В случае если один или несколько Уровней обслуживания, обозначенных в настоящем документе, не удовлетворены, Заказчик может уведомить об этом Менеджера SAP по работе с заказчиками.
- (c) Затем SAP оперативно определит основную причину или возможную основную причину проблемы и разработает план по ее исправлению.
- (d) В зависимости от ситуации, SAP может применить Зачетную неустойку по уровню обслуживания.
- (e) SAP освобождается от обязательств по выплате соответствующей Зачетной неустойки по уровню обслуживания, если анализ основных причин покажет, что несоблюдение соответствующего Уровня обслуживания допущено по причинам, зависящим от Заказчика.

## 4. Недостижение Уровня обслуживания

- (a) Зачетная неустойка по уровню обслуживания.
- (b) В случае подтверждения SAP права Заказчика на получение Зачетной неустойки по уровню обслуживания, SAP зачтет размер такой неустойки в счёт уменьшения оплаты Облачной услуги.
- (c) Прекращение действия. Если SAP не выполняет Соглашение об уровне обслуживания по Доступности системы для Продуктивной вычислительной среды на протяжении 3 (трех) последовательных месяцев.
